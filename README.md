## ⚠️ AI SAFETY & SECURITY WARNINGS

### Before Using ANY AI Assistant, Understand These Risks:

#### 🔴 **CRITICAL SECURITY CONCERNS**

1. **Prompt Injection Attacks**
   - AI assistants can be manipulated by malicious inputs
   - Attackers can trick the AI into revealing sensitive data
   - Your personal information, API keys, and credentials are at risk

2. **Data Privacy Risks**
   - All conversations may be logged and transmitted to third-party AI providers
   - Your messages, files, and commands could be stored indefinitely
   - Corporate/personal confidential information could leak

3. **Credential Exposure**
   - API keys and authentication tokens may be visible in logs
   - OAuth tokens can be intercepted or misused
   - Multi-platform integration = multiple attack vectors

4. **Dependency Vulnerabilities**
   - Outdated packages (like this repo) contain known security flaws
   - NPM/Node ecosystem has frequent critical vulnerabilities
   - 11,175 commits behind = missing critical security patches

#### 🟠 **OPERATIONAL RISKS**

1. **Unreliable AI Outputs**
   - AI assistants **hallucinate** false information
   - Can generate **harmful or biased** content
   - May execute **unintended system commands**

2. **Resource Consumption**
   - Always-on assistants consume significant CPU/RAM
   - Network bandwidth for constant AI API calls
   - Potential for runaway costs with paid AI services

3. **Platform Lock-In**
   - Dependency on specific AI providers (Anthropic, OpenAI)
   - Subscription requirements for full functionality
   - Service discontinuation risks

#### 🟡 **PRIVACY CONCERNS**

1. **Third-Party Data Sharing**
   - Your data passes through: WhatsApp, Telegram, Slack, Discord, AI providers
   - Each platform has different privacy policies
   - Data may be subject to government surveillance/subpoenas

2. **Metadata Exposure**
   - Timing, frequency, and patterns of AI usage tracked
   - Contact lists and social graphs may be accessible
   - Cross-platform correlation possible

3. **Permanent Records**
   - AI training data may include your conversations
   - Deletion requests often not honored
   - Data breaches could expose historical interactions

## 🔒 **RESPONSIBLE AI ASSISTANT USAGE**

If you still choose to use an AI assistant (from the upstream project), follow these guidelines:

### Security Best Practices:

1. **Never share sensitive data**
   - No passwords, API keys, or credentials
   - No personal identification numbers (SSN, passport, etc.)
   - No confidential business/medical information

2. **Audit what the AI can access**
   - Review all permissions and integrations
   - Limit filesystem and command execution access
   - Use sandboxed environments when possible

3. **Monitor AI behavior**
   - Check logs for unexpected activity
   - Review all executed commands before running
   - Set up alerts for unusual patterns

4. **Keep everything updated**
   - Update dependencies weekly
   - Monitor security advisories
   - Use `npm audit` / `pnpm audit` regularly

5. **Use dedicated accounts**
   - Separate AI assistant accounts from personal/work accounts
   - Dedicated API keys with minimal permissions
   - Separate messaging app accounts if possible

### Privacy Best Practices:

1. **Assume everything is logged**
   - Treat all AI conversations as public
   - Don't discuss private matters
   - Use end-to-end encryption where available

2. **Read privacy policies**
   - Understand data retention policies
   - Know your rights (GDPR, CCPA, etc.)
   - Request data deletion when possible

3. **Minimize data collection**
   - Disable unnecessary features
   - Opt out of analytics where possible
   - Use privacy-focused alternatives

### Ethical Considerations:

1. **Environmental impact**
   - AI models consume significant energy
   - Consider carbon footprint of constant AI usage

2. **Labor concerns**
   - AI training often involves exploited workers
   - RLHF (Reinforcement Learning from Human Feedback) data labeling conditions

3. **Misinformation risks**
   - Verify AI-generated facts independently
   - Don't rely on AI for critical decisions
   - Understand AI limitations and biases
